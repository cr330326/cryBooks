# 大语言模型论文深度摄入

## 核心结论

这组论文形成一条清晰技术链：Transformer 提供基础架构；GPT-4、Llama 3、Qwen、DeepSeek 和 Apple Intelligence 展示不同模型族的训练、对齐、部署和评估路线；ReAct、GraphRAG 和 DeepSeek-R1 则把模型能力推向工具使用、知识整合和强化学习驱动的推理。

## 架构起点：Transformer

`Attention Is All You Need` 以自注意力替代循环结构，奠定了后续 LLM 的核心架构。它把序列建模中的依赖聚合交给 self-attention，并用堆叠的编码器/解码器、前馈层和位置编码构成模型主体。

## 大模型能力与系统卡

`GPT-4 Technical Report` 把 GPT-4 描述为可接受图像和文本输入、输出文本的大规模多模态模型，并强调专业/学术 benchmark 上的人类级表现、安全挑战和系统卡风险评估。

`Apple Intelligence Foundation Language Models` 体现端云协同路线：小模型面向本地设备效率，大模型面向 Private Cloud Compute，同时强调准确、负责和任务适配。

## 开源模型族路线

`The Llama 3 Herd of Models` 展示一组支持多语言、代码、推理和工具使用的模型族，最大 405B dense Transformer，长上下文可达 128K。

`Qwen Technical Report` 和 `DeepSeek LLM Scaling...` 展示中文/多语言开源模型训练路线，主题包括预训练数据、tokenization、架构、训练、长上下文扩展、scaling laws 和基础设施。

## 推理强化学习路线

`DeepSeek_R1` 的核心在于用强化学习激励推理能力，区分 R1-Zero 和 R1，强调无需先 SFT 的大规模 RL 可以涌现复杂推理行为。

OpenAI 竞赛推理报告则从竞技编程角度说明大推理模型通过 RL 获得复杂测试时推理策略，逐步减少对人工设计推理策略的依赖。

## Agent 与工具使用

`REACT` 提出把 reasoning 和 acting 结合起来，让模型在推理、行动、观察之间循环。这是 Agent Loop 的研究基础，也与 [[AI Agent Runtime]] 中的工具执行闭环直接相连。

`Generative Agents` 展示观察、规划、反思构成的可信行为模拟架构，是多 Agent 社会模拟和长时记忆/计划的代表。

## RAG 与知识库

`GraphRAG` 针对传统向量 RAG 难以回答全局 sensemaking 问题的缺陷，使用图结构和分层摘要帮助模型获得对整个语料的全局理解。这与当前 LLM Wiki 方法高度相关：二者都强调“先构建中间知识结构，再查询综合”。

## 对本 Wiki 的启发

- LLM Wiki 可以视为人工维护/Agent 维护的 GraphRAG 中间层。
- Agent 深度摄入时应区分模型架构、训练数据、对齐方法、推理方法、评估体系和应用运行时。
